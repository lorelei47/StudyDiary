## 面向对象的程序设计

#### 理解对象

> 属性类型

```
数据属性：包含一个数据值的位置。这个位置上可以读取和写入值。有四个描述其行为的特性
Configurable:示能否通过 delete 删除属性从而重新定义属性，默认true
Enumerable:能否通过 for-in 循环返回属性，默认true
Writable:示能否修改属性的值，默认true
Value:就是这个属性的数据值

可以通过Object.defineProperty()来更改上述前三种特性的值
var person = {}; 
Object.defineProperty(person, "name", { 
 writable: false, 
 value: "Nicholas" 
}); 
alert(person.name); //"Nicholas" 
person.name = "Greg"; 
alert(person.name); //"Nicholas"
将person的name属性改成只读

访问器属性：只包含getter和setter函数
```

#### 创建对象

> 工厂模式

```
Object构造函数可以创建单个对象，但如果需要创建多个对象则会产生重复代码。这时候就诞生了一种模式，工厂模式。
工厂函数：
function createPerson(name, age, job){ 
 var o = new Object(); 
 o.name = name; 
 o.age = age; 
 o.job = job; 
 o.sayName = function(){ 
 alert(this.name); 
 }; 
 return o; 
} 
var person1 = createPerson("Nicholas", 29, "Software Engineer"); 
var person2 = createPerson("Greg", 27, "Doctor");
抽象了对象细节出来，封装到一个函数内。
```

> 构造函数模式

```
function Person(name, age, job){ 
 this.name = name; 
 this.age = age; 
 this.job = job; 
 this.sayName = function(){ 
 alert(this.name); 
 }; 
} 
var person1 = new Person("Nicholas", 29, "Software Engineer"); 
var person2 = new Person("Greg", 27, "Doctor");
构造函数模式与工厂模式区别：
1) 没有显式地创建对象；
2) 直接将属性和方法赋给了 this 对象；
3) 没有 return 语句。
4) 函数名第一个字母大写
5) 创建实例时必须使用new操作符
构造函数实际执行步骤如下；
(1) 创建一个新对象；
(2) 将构造函数的作用域赋给新对象（因此 this 就指向了这个新对象）；
(3) 执行构造函数中的代码（为这个新对象添加属性）；
(4) 返回新对象。
上述定义方式是将构造函数定义在Global对象中。

构造函数的使用
// 当作构造函数使用
var person = new Person("Nicholas", 29, "Software Engineer"); 
person.sayName(); //"Nicholas" 
// 作为普通函数调用
Person("Greg", 27, "Doctor"); // 添加到 window 
window.sayName(); //"Greg" 
// 在另一个对象的作用域中调用
var o = new Object(); 
Person.call(o, "Kristen", 25, "Nurse"); 
o.sayName(); //"Kristen"
```

> 原型模式

```
每个函数都有一个原型属性prototype，是一个指针，指向一个对象。而这个对象的用途是包含可以由特定类型的所有实例共享的属性和方法。
例如：
function Person(){ 
} 
Person.prototype.name = "Nicholas"; 
Person.prototype.age = 29; 
Person.prototype.job = "Software Engineer"; 
Person.prototype.sayName = function(){ 
 alert(this.name); 
}; 
var person1 = new Person(); 
person1.sayName(); //"Nicholas"
```
